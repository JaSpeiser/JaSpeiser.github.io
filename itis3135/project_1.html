Overview

  • Purpose: An application for organizing board game events
  • Intended Users: Friends and family of client
  • Overview: A place to learn about client's board game events and sign up for upcoming events

Wireframe


Sitemap


Pages

Home

  • Purpose: A landing page that helps users navigate to other pages
  • Audience: Users visiting the website for the first time
  • Content: Images of previous board game nights and links to other pages
  • Data: N/A
  • Actions: Links that will navigate to other pages

About/Info

  • Purpose: To educate to user about client's events
  • Audience: Users looking to learn more about what board game night is
  • Content: Informative paragraphs
  • Data: N/A
  • Actions: N/A

Previous Events

  • Purpose: To show the user what board game night looks like and what they can expect
  • Audience: Users wanting to know what client's events are like
  • Content: Images of previous events accompanied by short paragraphs about said events
  • Data: N/A
  • Actions: N/A

Games

  • Purpose: To show users what games have already been played and what games haven't
  • Audience: Users who want to know what type of board games are played at events and what board games will be played in the future
  • Content: List of board games that have been played next to list of board games that haven't
  • Data: N/A
  • Actions: Listed board games will be hyperlinks that navigate to each game's official website

Sign Up

  • Purpose: For users to sign up to take part in future board game nights
  • Audience: Users that have viewed all other pages and want to take part in future events
  • Content: A submittable form that gathers user information and a table showing users that have already signed up for the next event as well as how many spots are available
  • Data: First name, last name, email, and phone number. Data validation needed to ensure user has not already signed up, each entry matches the expected type (i.e. email is an actual email), and there are enough spots left for the user to sign up
  • Actions: Submit button will update the table with entered form data when pressed

Dynamic Functionality

Sign up page will have a table that shows users who have already signed up and empty spots available. The table will be a fixed size, so spots available will be denoted by empty rows. Upon submission of the form, the table will update to include the information that was just submitted.
